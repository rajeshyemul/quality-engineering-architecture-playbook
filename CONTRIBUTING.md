# Contributing to This Playbook

This is a living document. As patterns evolve, systems scale, and new challenges emerge, this playbook should evolve with them.

## How to Contribute

**Adding a new pattern:** Create a new file in the appropriate section directory. Follow the existing format: context, problem statement, solution, implementation, trade-offs.

**Updating an existing pattern:** Open a PR with the change and rationale. Include what changed in practice that motivated the update.

**Adding examples:** Real examples from production systems (anonymized as needed) are more valuable than theoretical examples. Add them.

## Document Format

Each document should include:
- **Context** — what kind of system or situation does this apply to?
- **Problem** — what specific challenge does this address?
- **Solution** — the pattern, practice, or model
- **Implementation** — how to actually do it, with code examples where relevant
- **Trade-offs** — what does this cost? When does it not apply?
- **Links** — to related documents in the playbook

## Review Standard

Before merging a contribution:
- [ ] Is the writing at the appropriate depth level for its section?
- [ ] Does it connect to the surrounding documents?
- [ ] Are code examples tested and accurate?
- [ ] Does it add real value, or duplicate existing content?
